# 车辆使用寿命预测

本项目来自零碳科技解决方案创新大奖赛“妙算榜”赛题，参赛单位为金陵科技学院。项目目标是基于新能源汽车运行数据，完成车辆使用寿命与电池健康状态预测，为二手车估值、电池回收和车辆维护提供数据支撑。

答辩材料中给出的核心思路是：先对车辆数据进行清洗和筛选，再基于 SOC 区间和安时积分法估算电池容量，进一步计算 SOH，最后利用 EMD 分解与机器学习模型预测未来寿命趋势。

## 项目背景

新能源汽车的剩余寿命预测与电池健康状态直接相关。电池健康度会影响二手车估值、退役电池回收和后续维护策略。传统固定公式难以处理温度、充放电习惯、里程增长等动态因素，因此需要结合数据处理与机器学习模型进行预测。

项目答辩 PPT 中总结了三个难点：

- 数据复杂：电池、电压、电流、SOC、里程和环境因素需要融合分析。
- 动态耦合：电池老化与温度、充放电行为之间存在非线性关系。
- 部署要求：模型既要有较高精度，也要便于后续解释和应用。

## 方法概览

本项目先筛选并修正 SOC、充电状态和里程等数据，再选取 SOC 50% 到 80% 区间进行容量估算。该区间中电池状态相对稳定，利用安时积分法估算容量时可靠性更高。

容量序列经过异常值处理和中值滤波后，用于计算 SOH。预测阶段将时间序列转化为监督学习问题，并比较 GPR、SVR、LSTM、XGBoost 以及 EMD 分解后的组合模型。答辩方案中最终采用 EMD-XGBoost-GPR 思路：高频噪声部分使用 XGBoost 建模，低频残差部分使用 GPR 建模，再重构得到最终预测结果。

## 脚本分类

| 目录 | 作用 | 代表脚本 |
| --- | --- | --- |
| `src/preprocessing/` | 原始数据清洗、SOC 与充电状态筛选、异常值处理 | `0精简数据.py`、`1筛选.py`、`2去除SOC和充电状态异常值.py` |
| `src/feature_engineering/` | SOC 区间筛选、容量积分、里程-容量提取、SOH 计算、滤波 | `32电池容量积分.py`、`5_SOH计算.py`、`42中值滤波.py` |
| `src/modeling/` | GPR、SVR、LSTM、XGBoost、EMD 组合预测 | `51_EMD-XGboost-GPR.py`、`51_GPR.py`、`61_里程预测.py` |
| `src/matlab/` | 时间排序辅助脚本 | `time_sort.m` |

## 推荐流程

原始代码按实验过程命名，建议按以下顺序理解：

1. 精简并筛选原始数据：`0精简数据.py`、`1筛选.py`
2. 去除 SOC 和充电状态异常：`2去除SOC和充电状态异常值.py`
3. 筛选 SOC 50% 到 80% 区间：`31筛选SOC区间.py`
4. 计算容量并提取里程容量关系：`32电池容量积分.py`、`33提取_里程-容量.py`
5. 处理容量异常并滤波：`41去除异常值(容量).py`、`42中值滤波.py`
6. 计算 SOH：`5_SOH计算.py`
7. 比较预测模型：`51_GPR.py`、`51_SVR 预测.py`、`51_EMD-XGboost-GPR.py`
8. 做未来里程与寿命趋势预测：`61_里程预测.py`、`61_预测两年otometer.py`

## 环境依赖

建议使用 Python 3.9 或以上版本。

```bash
pip install -r requirements.txt
```

部分脚本使用 TensorFlow/LSTM，安装时间会较长。如果只运行传统机器学习模型，可以先安装 `pandas`、`numpy`、`scikit-learn`、`xgboost`、`matplotlib`、`scipy` 和 `EMD-signal`。

## 数据说明

本打包版本只整理算法脚本和项目说明，没有包含原始车辆数据。请将比赛数据或本地处理数据放入 `data/` 目录，并根据脚本中的文件路径修改输入输出位置。

建议不要把大体积原始数据、模型文件和中间结果直接上传 GitHub，可使用 Release、网盘或本地路径管理。
